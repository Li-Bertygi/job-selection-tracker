# job-selection-tracker

## 1. サービス概要

本サービスは、企業ごとに異なる就職活動の選考プロセスを一元的に管理し、  
応募状況・選考ステータス・面接日程・メモを構造的に整理できる選考管理システムです。

---

## 2. 課題定義

就職活動を進める中で、複数企業の選考を同時に管理する際に以下のような課題を感じました。

- 企業ごとに選考フローが異なり、進捗管理が煩雑になる
- 面接日程や結果通知日、提出期限などの情報が分散し、管理が困難である
- 志望動機や面接での回答、振り返りなどの情報を企業ごとに整理できない
- 選考が増えるにつれて、情報の抜け漏れやスケジュールの重複が発生しやすい

これらの課題は、単なるメモやスプレッドシートでは構造的に管理することが難しく、  
「選考プロセスそのもの」を管理できる仕組みが必要だと考えました。

---

## 3. MVP範囲

本プロジェクトでは、上記の課題を解決するために、以下の機能をMVPとして実装しました。

- ユーザー認証（ログイン・JWTベース認証）
- 企業情報および応募情報の登録・管理
- 応募ごとの選考ステージ（書類選考、面接など）の管理
- 面接日程や結果通知日のスケジュール管理
- 応募ごとのメモ（志望動機、面接内容、振り返りなど）の記録
- 現在の選考ステータスの可視化

また、初期段階では機能を過度に拡張せず、  
「選考プロセスを構造的に管理する」というコア機能に集中することで、  
システムの設計とデータ構造の妥当性を重視しました。
